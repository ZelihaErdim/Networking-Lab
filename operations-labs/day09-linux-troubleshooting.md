# Day 9 — Linux Troubleshooting

## Learn
- `journalctl` shows system logs
- `systemctl` manages services
- `systemctl status ssh` checks whether SSH is running
- `journalctl -xe` shows recent log errors

## Practice Tasks
- Check whether key services are running
- Review logs for failures
- Practice identifying why a service is down

## Reflection Questions
- How do you tell if a service is failing because of configuration or runtime issues?
- What logs help you narrow down the problem fastest?
